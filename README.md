予防接種情報検索アプリ 要件定義書
version 1.0

　　 作成日：2026-1-6
作成者：大向 なるみ


1. はじめに
1.1 本書の目的
本書は、予防接種に関する情報を利用者が事前に把握できる Web アプリケーションの開発に向け、要件を整理・定義することを目的とした要件定義書である。
利用者が医療機関ごとの予防接種費用の目安を把握しやすくすることで、情報収集に伴う負担の軽減および医療機関への不要な問い合わせの削減を目指す。

1.2 本書の位置づけ
本書は、システム開発における要件定義フェーズの成果物であり、後続工程である基本設計および詳細設計のインプット資料として使用する、

1.3 対象読者
本書の対象読者は以下を想定する。
本システムの企画・開発・運営に関わる関係者
基本設計・実装を担当する開発者


2. 背景・課題
2.1 現状の課題
予防接種を希望する利用者にとって、医療機関ごとの接種費用や対応内容を事前に把握することは容易ではない。特に以下の点が課題となっている。
医療機関ごとに予防接種費用が異なり、価格にばらつきがある
事前に費用や対応内容を比較できる手段が乏しい
電話による問い合わせが主な確認手段となっており、心理的・時間的な負担が大きい
その結果、利用者は十分な情報を得られないまま医療機関を選択せざるを得ない状況となっている。


2.2 既存手段の問題点
現状、予防接種に関する情報収集には以下のような手段が用いられているが、いずれも十分とは言えない。
Google Map 等の地図サービスでは、医療機関の位置や基本情報は確認できるものの、予防接種費用の比較はできない
医療機関の公式サイトにおいて、予防接種費用が未掲載、もしくは情報が不十分なケースが多い
自治体が提供する予防接種情報と、個別医療機関が提供する情報が分断されており、 利用者が横断的に情報を把握しにくい


これらの問題により、利用者は複数の情報源を行き来する必要があり、情報収集に大きな負担が生じている。


3. 本システムの目的
3.1 目的
本アプリは、予防接種を検討している利用者が、医療機関ごとの予防接種費用の目安を事前に把握できるよう支援することを目的とする。
これにより、医療機関への不要な電話問い合わせを減らし、利用者および医療機関双方の負担軽減を図る。

3.2 本システムで解決しないこと（重要）
本アプリは情報提供を目的としたシステムであり、以下の事項については対象外とする。
医療行為に関する予約、斡旋、または契約行為
医学的な判断、診断、助言
掲載される予防接種費用の正確性や最新性の保証



4. 想定ユーザー
4.1 一般利用者
本アプリの主な利用者は、予防接種に関する情報を必要とする一般利用者とする。
 具体的には、以下の利用者層を想定する。
子育て世代（乳幼児・小児の定期接種や任意接種を検討する保護者）
高齢者（インフルエンザ等の定期的な予防接種を受ける利用者）
持病があり、医師の指示により定期的な予防接種が必要な利用者

4.2 医療機関（将来）
将来的な拡張として、医療機関が自ら予防接種に関する情報を更新・管理できる利用形態を想定する。
予防接種費用や対応ワクチン情報を自ら更新したい医療機関
正確な情報提供により、問い合わせ対応の負担軽減を図りたい医療機関


なお、本フェーズでは医療機関向けの管理機能は対象外とする。


5. 利用シナリオ（ユースケース）
5.1 利用者側
一般利用者は、以下の流れで本アプリを利用することを想定する。
利用者は、地域およびワクチン名を指定して検索を行う
検索結果として、医療機関ごとの予防接種費用の目安、所在地までの距離等を確認する
詳細な情報が必要な場合は、医療機関の公式サイトへ遷移する


本アプリ内での予約や問い合わせは行わず、最終的な判断・連絡は利用者自身が行うものとする。


5.2 管理者側（将来）
将来的な拡張として、管理者（運営者）による以下の対応を想定する。
掲載情報の修正および一時的な非表示対応
利用者からの通報内容の確認および対応


なお、本フェーズでは管理者向けの操作画面や管理機能の実装は対象外とする。




6. 機能要件
6.1 検索機能
利用者は、以下の条件を指定して医療機関を検索できるものとする。
地域による検索
ワクチン種別による検索


複数条件を組み合わせた検索を可能とし、利用者が目的に応じた情報を効率的に取得できるようにする。


6.2 表示機能
検索結果には、以下の情報を表示するものとする。
予防接種費用の価格帯（目安）
医療機関名
利用者の現在地から医療機関までの距離


なお、距離表示については、将来的に Google Maps 等の地図サービスとの連携を想定するが、本フェーズでは簡易的な距離表示に留める。

6.3 口コミ・情報提供（段階的）
将来的な拡張として、利用者による情報提供機能を段階的に導入することを想定する。
利用者による予防接種費用に関する情報投稿
投稿内容に対する管理者による承認・公開制御


なお、本フェーズでは情報投稿機能の実装は対象外とする。


7. 非機能要件
7.1 信頼性
本アプリに掲載される口コミや価格情報は、あくまで参考情報として扱うものとする。
利用者に対しては、最終的な内容確認や判断は医療機関へ直接行う必要がある旨を明示する。

7.2 法令・ガイドラインへの配慮
本アプリは、医療広告ガイドラインをはじめとする関連法令およびガイドラインに配慮して運用する。
誇張表現や特定の医療機関・医療行為への誘導となる表示を行わない
客観性を欠く表現や誤解を招く表現を排除する


7.3 個人情報の取り扱い
利用者による情報投稿や利用にあたっては、個人情報の保護に十分配慮する。
投稿は匿名で行えるものとする
個人が特定される情報は保存・管理しない


7.4 運用
本アプリは、継続的な情報更新を前提として運用する。
予防接種費用の価格改定や季節変動に対応できる運用とする
医療機関または利用者からの情報削除・修正依頼に対応するためのフローを整備する



8. 制約条件
8.1 技術的制約
本システムは、初期フェーズにおいて以下の技術的制約のもとで構築する。
初期提供形態は Web アプリケーションとする
外部 API（地図情報等）については、必要性や運用状況を踏まえ、段階的に導入する


8.2 運用上の制約
本システムの運用にあたっては、以下の制約を設ける。
掲載される情報の正確性や最新性について、保証は行わない
医療行為、診療判断、治療方針の決定には一切関与しない


9. 収益モデル（構想）
9.1 想定モデル
将来的な運営継続を見据え、以下の収益モデルを構想する。
医療機関向け管理機能の月額課金提供
医療機関向けのデータ分析サービス提供（将来検討）


9.2 実装範囲外の扱い
本フェーズでは、収益化に関する機能の実装は行わない。
収益モデルはあくまで構想段階とし、要件定義および設計の対象外とする。
